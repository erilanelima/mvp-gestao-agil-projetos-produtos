# MVP - Gestão Ágil de Projetos e Produtos

## Objetivo

Este repositório contém a documentação e os artefatos desenvolvidos para a construção de um **MVP (Minimum Viable Product)**, aplicando os conceitos estudados nas disciplinas:

* Engenharia de Requisitos e Gestão Ágil de Produtos
* Gestão Ágil de Projetos
* Design e Prototipação da Interface do Usuário

O projeto contempla a ideação, planejamento, definição do backlog, planejamento da primeira sprint e prototipação de interfaces para um produto de software desenvolvido em um cenário de negócio hipotético.

---

# Escopo da Entrega

## 1. Contexto do Negócio

Definição do cenário hipotético contendo:

* Descrição do produto proposto;
* Identificação dos stakeholders;
* Identificação dos usuários e clientes;
* Composição do Time Scrum;
* Papéis, responsabilidades e habilidades da equipe.

---

## 2. Lean Inception e MVP Canvas

Realização completa de uma Lean Inception para definição do MVP, contemplando:

* Visão do produto;
* Objetivos de negócio;
* Personas;
* Jornada do usuário;
* Brainstorm de funcionalidades;
* Priorização;
* Sequenciamento das entregas;
* MVP Canvas final.

Toda a documentação foi construída utilizando o Miro.

### Arquivo de Referência

`canvas-url.txt`

---

## 3. Backlog do Produto

Construção do backlog inicial do produto contendo:

### Estrutura Hierárquica

* Épicos
* Features
* Histórias de Usuário
* Enablers (quando necessário)

### Informações Incluídas

* Descrição dos itens;
* Prioridade;
* Critérios de aceitação;
* Estimativas;
* Relacionamentos entre itens.

### Governança do Backlog

Também estão definidos:

* Definition of Ready (DoR)
* Definition of Done (DoD)

Incluindo pelo menos um requisito não funcional.

### Arquivo

`product-backlog.pdf`

---

## 4. Planejamento da Primeira Sprint

Seleção dos itens prioritários do backlog para compor a Sprint 1.

Cada item selecionado possui:

* História de usuário detalhada;
* Critérios de aceitação;
* Estimativa em Story Points;
* Atendimento ao Definition of Ready.

### Arquivo

`sprint-backlog.pdf`

---

## 5. Protótipos de Interface

Desenvolvimento de wireframes de baixa fidelidade utilizando o Figma para as funcionalidades previstas na primeira sprint.

### Localização

```text
wireframes/
```

Os protótipos foram exportados como imagens.

Vídeo de demonstração da solução apresentando:

* Contexto do negócio;
* Lean Inception;
* MVP Canvas;
* Backlog do Produto;
